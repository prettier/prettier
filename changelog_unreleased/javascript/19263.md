#### Keep trailing comments inside the parentheses of a sequence expression (#19263 by @sarathfrancis90)

A trailing comment just before the closing parenthesis of a parenthesized
`SequenceExpression` used to print outside the parentheses and drift past the
`;` on each reformat. It now stays inside the parentheses, extending the
arrow-body fix (#19253) to variable declarators, expression statements,
return statements, and assignments.

<!-- prettier-ignore -->
```jsx
// Input
const x = (a, b, c /* comment */);

// Prettier stable
const x = (a, b, c) /* comment */;

// Prettier main
const x = (a, b, c /* comment */);
```
