#### Preserve `yield` semantics during range formatting (#16167 by @STRd6)

When the selected range fell entirely inside a generator function's body, Prettier
sliced the text and reparsed it as a standalone program. With no surrounding
generator context, `yield` parses as an identifier and `yield [1, 2]` becomes a
member expression `yield[(1, 2)]`. Range formatting now widens the slice to the
nearest enclosing source element that contains the binding generator function.

<!-- prettier-ignore -->
```jsx
// Input (range covers only the `yield` line)
class ExampleIterator {
  *[Symbol.iterator]() {
    yield [1, 2];
  }
}

// Prettier stable
class ExampleIterator {
  *[Symbol.iterator]() {
    yield[(1, 2)];
  }
}

// Prettier main
class ExampleIterator {
  *[Symbol.iterator]() {
    yield [1, 2];
  }
}
```
