#### Honor `--range-start`/`--range-end` with `--config-precedence prefer-file` (#19241 by @xfocus3)

Editor options (`--cursor-offset`, `--range-start`, `--range-end`) describe a single
invocation rather than a project's code style, so they can't be set in a config file.
Previously `--config-precedence prefer-file` ignored them whenever a config file was found,
which broke editor integrations passing a range. They are now kept while other CLI options
are still ignored, as documented.

<!-- prettier-ignore -->
```console
$ cat .prettierrc
{}

$ cat file.js
foo(  1  );
bar(  2  );

# Prettier stable
$ prettier --config-precedence prefer-file --range-start 0 --range-end 11 file.js
foo(1);
bar(2);

# Prettier main
$ prettier --config-precedence prefer-file --range-start 0 --range-end 11 file.js
foo(1);
bar(  2  );
```
