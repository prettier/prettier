#### Warn when a glob matches only ignored files (#19566 by @ahonnecke)

When a glob pattern matches files that are all filtered out by an ignore file (such as `.prettierignore`), Prettier now prints a warning instead of silently doing nothing. Explicitly passed directories (e.g. `prettier .`) never trigger the warning, so the common `prettier --check .` usage is unaffected.

<!-- prettier-ignore -->
```sh
prettier "ignored/*.js"

# Prettier stable
# (no output, exit code 0)

# Prettier main
[warn] All files matching the pattern "ignored/*.js" are ignored.
```
