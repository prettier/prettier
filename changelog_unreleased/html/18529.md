#### Break the closing tag of an empty element to a new line when the opening tag breaks (#18529 by @rkdfx)

When an empty HTML/Vue/Angular element's opening tag breaks across multiple lines (long attributes or `--single-attribute-per-line`), the closing tag now lands on its own line, mirroring the behavior of non-empty elements. Self-closing and void elements (`<img>`, `<br>`, `<input>`, `<MyComp />`) are unaffected.

<!-- prettier-ignore -->
```html
<!-- Input -->
<div class="some-class" data-placeholder="some data"></div>
<div class="some-class" data-placeholder="some data">test</div>

<!-- Prettier stable, options: --bracket-same-line --single-attribute-per-line -->
<div
  class="some-class"
  data-placeholder="some data"></div>
<div
  class="some-class"
  data-placeholder="some data">
  test
</div>

<!-- Prettier main, options: --bracket-same-line --single-attribute-per-line -->
<div
  class="some-class"
  data-placeholder="some data">
</div>
<div
  class="some-class"
  data-placeholder="some data">
  test
</div>
```
