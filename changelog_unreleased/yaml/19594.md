#### Print YAML anchors before tags (#19594 by @paranoa233)

Prettier now prints YAML anchors before tags, making it clearer that an alias refers to the tagged value associated with the anchor.

<!-- prettier-ignore -->
```yaml
# Input
One: &anchor !custom Value
Two: *anchor

# Prettier stable
One: !custom &anchor Value
Two: *anchor

# Prettier main
One: &anchor !custom Value
Two: *anchor
```
