#### Preserve multiline inline code in list items when `proseWrap` is `preserve` (#18990 by @bduffany)

Fix an issue that repeated formatting could change the semantic spaces inside multiline inline code when the code span starts on a list item line and closes on a later continuation line.

<!-- prettier-ignore -->
```markdown
<!-- Input -->
- `3
        baz`

<!-- Prettier stable -->
- `3
      baz`

<!-- Prettier main -->
- `3
        baz`
```
