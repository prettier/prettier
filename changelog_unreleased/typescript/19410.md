#### Preserve comment position after `=` in type alias declarations (#19410 by @WhoamiI00)

When parsing strips single-type unions or parenthesized types (e.g. `type A = /* x */ | T`), comments authored between `=` and the type annotation could be moved in front of `=`. They now stay where the source put them. Comments authored before the `=` keep their original position.

<!-- prettier-ignore -->
```ts
// Input
type A =
  /* 1 */ | /* 2 */ (
    /* 3 */ | /* 4 */ {
        key: string;
      }
  );

// Prettier stable
type A /* 2 */ = /* 1 */ /* 3 */ /* 4 */ {
  key: string;
};

// Prettier main
type A = /* 1 */ /* 2 */ /* 3 */ /* 4 */ {
  key: string;
};
```
