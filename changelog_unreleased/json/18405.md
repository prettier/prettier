#### Preserve number and string representation in `json-stringify` parser (#18405 by @fisker)

Previously, we printed numbers and strings with `JSON.stringify()`, but it loses the original representation sometimes. Eg: big/small numbers are wiped, special characters get unescaped. Technically, it doesn't change the value, but that's something the user expected. Prettier main keeps the original representation for them.

// Input
[
  "\u00FF",
  1e9999,

  -9223372036854775809,
  1e3,
  {
    1e3: 1,
    1e999999999999999999999999999999: 1,
    1: 1
  }
]
