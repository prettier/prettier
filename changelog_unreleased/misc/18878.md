#### Clarify trailing `// prettier-ignore` behavior in docs (#18878 by @alubbe)

Updated the ignore documentation to explicitly describe both JavaScript forms:

- Own-line `// prettier-ignore` applies to the next AST node.
- End-of-line `// prettier-ignore` applies to the preceding node on that line.

Added matching examples in both the source docs and versioned stable docs so the behavior is documented consistently.
